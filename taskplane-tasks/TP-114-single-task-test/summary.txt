Runtime V2 keeps Taskplane’s durable execution model while replacing the fragile runtime control plane. Core task artifacts such as PROMPT.md, STATUS.md, .DONE, dependency waves, and .pi/batch-state.json remain authoritative, but orchestration no longer depends on TMUX sessions, session name coupling, or sidecar-file polling between sibling processes.

The architecture introduces deterministic ownership boundaries: the extension owns an engine process, the engine owns lane-runner and merge hosts, lane-runner owns worker/reviewer hosts, and each agent-host owns its Pi child process. This creates direct parent-child IPC paths for live telemetry and lifecycle management, while durable files remain the long-term recovery and observability record.

Runtime V2 also formalizes mailbox-based steering and normalized runtime storage under .pi/runtime/{batchId}, including per-agent manifests, event logs, stderr tails, exit summaries, lane snapshots, and a registry snapshot. The design goal is stable, resumable, and observable orchestration where correctness does not depend on terminal infrastructure, and where /orch is the authoritative execution path even for single-task runs.
